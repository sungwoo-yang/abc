Additional Notes

  • Here's how you deal with the exception if there is no more memory available. Since you're using the ObjectAllocator, you won't be calling new for any nodes. (You won't be calling delete either.) However, you will be using placement new to construct the data. Here's an example of how you might do it: 
    try 
{
  Node *mem = reinterpret_cast<Node *>(objAllocator_->Allocate()); // Allocate memory for the object.
  node = new (mem) Node(Item); // Construct the object in the memory.
}
catch (const OAException &e)
{
  throw(SortedListException(SortedListException::E_NO_MEMORY, e.what()));
}
  • Don't forget to call the destructor explicity on the node when you remove it from the list. For example: 
    some_node->~Node();             // Call the object's destructor.
objAllocator_->Free(some_node); // Free the object's memory.

  • If you're implementing the subscript operator, you don't have to do any bounds-checking and throw an exception. This is to simplify things. The driver will not give you an invalid subscript.

  • With the exception of the two sort methods and the extra credit iterator methods, you have implemented all of the methods in the SortedList class in CS170 and other courses. That's what makes this assignment straight-forward. You're really only implementing two new methods. Granted, one of those methods is fairly complex.

    Also, using the ObjectAllocator instead of calling new is trivial because there is only one method (and one line of code) in the entire class that should be allocating nodes. (You should know which one.)
